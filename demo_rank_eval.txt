# Demo Ranking Evaluation API (_rank_eval)

# https://www.elastic.co/guide/en/elasticsearch/reference/current/search-rank-eval.html

# The ranking evaluation API allows to evaluate the quality of ranked search results over a set of typical search queries. Given this set of queries and a list or manually rated documents, the _rank_eval endpoint calculates and returns typical information retrieval metrics like mean reciprocal rank, precision or discounted cumulative gain.

# Main goals:
# - facilitate query development and testing in Dev
# - enable long-term quality testing of existing search systems in Prod
# - provide building blocks for more sofisticated operations, e.g. (semi)-automatic tuning of boost, similarity parameters etc...

# --> on to the demo ...


# We are using a small subset of the english wikipedia corpus
GET /enwiki_rank/_count

GET /enwiki_rank/_mapping

GET /enwiki_rank/_doc/5119376

# subset selected to include rated documents from Wikimedia labs
# "discernatron" project: http://discernatron.wmflabs.org

# Public dataset of queries and ratings for wikipedia pages
# https://discernatron.wmflabs.org/scores/all?json=1

# the data looks like this:
# | query | title           | score  | num_scores |...
# +-------|-----------------|--------|------------|------------
# | JFK	  | John F. Kennedy	| 3.0000 | 4          |...
# | JFK   | JFK (film)	    | 2.2500 | 4          |...
# | JFK   | Kennedy family	| 1.7500 | 4          |...
# [...]

# 32 user queries, approx. 1800 ratings
# scores: 0 = irrelevant
#         1 = Maybe Relevant
#         2 = Probably Relevant
#         3 = Relevant

# - user is looking for info about John F. Kennedy (information need)

# Step 1.2: change field "text" -> "title"

{
  "_source": "title", 
  "query" : {
    "match" : {
      "all" : "JFK"
    }
  }
}

GET enwiki_rank/_search?filter_path=hits.total.value,hits.hits._source.title&q=all:JFK
